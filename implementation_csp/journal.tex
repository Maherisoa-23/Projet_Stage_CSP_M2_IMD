\section*{Journal des evolutions}
\addcontentsline{toc}{section}{Journal des evolutions}

Ce document evolue avec le projet. Pour faciliter le suivi, les blocs
\fcolorbox{primary}{blue!5}{\color{primary}\footnotesize\bfseries\textsc{Ajout du AAAA-MM-JJ}}
marquent dans le texte les contenus ajoutes a chaque etape.

\begin{center}
\begin{tabular}{lp{11cm}}
  \toprule
  Date & Contenu ajoute \\
  \midrule
  Avril 2026 &
    Version initiale : modele CSP (PyCSP3 + ACE), reconstruction 3D, validation
    xTB en single-run, infrastructure de batch (\texttt{batch\_main}, \texttt{batch\_all}),
    viewer HTML par taille et rapport global. \\
  Avril 2026 (mi-mois) &
    Validation multi-runs (\texttt{-{}-n-runs N}), classification de stabilite
    (6 classes), \texttt{aggregate\_runs.py}, viewer enrichi (badges, barre de
    dispersion SVG), section ``Couverture CSP'' (partition entre les 8 configs). \\
  1\textsuperscript{er} mai 2026 &
    Validation par \textbf{dynamique moleculaire courte} (xtb -{}-md + opt) selon le
    protocole des chimistes. Pattern \emph{Strategy} pour decoupler les methodes de
    validation. \texttt{aggregate\_md.py} additif. Refonte du dashboard en
    \textbf{MD-only}, KPIs uniformes, mini-tableau MD par config en haut de chaque
    \texttt{view.html}, section ``Couverture CSP'' alignee sur le verdict MD. \\
  Mai 2026 (mi-mois) &
    \textbf{MD devient la methode par defaut} (\texttt{-{}-method multi-runs} pour
    l'ancien comportement). \textbf{Table de voisinage regeneree avec MD} :
    T(5) 92$\to$63, T(6) 418$\to$280, T(7) 699$\to$367 (--41\%, faux positifs
    plats elimines). Drapeau \texttt{-{}-count-hexagon} : la solution tout-hexagones
    est desormais \textbf{exclue par defaut} des solutions CSP. Script
    \texttt{count\_all.py} : compte les solutions CSP sur les 8 configs sans xTB.
    Reconciliation des agregateurs (drop des entrees \texttt{data.json} obsoletes,
    preservation des blocs \texttt{md\_validation} lors d'un re-run multi-runs).
    Affichage ``Aucune solution CSP'' dans le viewer pour les molecules entierement
    gelees. \texttt{utils/types.py} renomme \texttt{csp\_types.py} (fix shadowing
    stdlib). \textbf{Documentation modulaire} : ce document est splitte en
    \texttt{implementation\_csp/} (preamble + sections + bibliographie). \\
  6 mai 2026 &
    \textbf{Determinisme byte-pour-byte du test de planarite}. Deux sources de
    non-determinisme corrigees : (1) iteration sur \texttt{set[str]} dans
    \texttt{reconstruction/assembler.py} -- l'ordre des atomes variait entre
    processus via \texttt{PYTHONHASHSEED}, donc xTB voyait des inputs
    permutes ; fixe par \texttt{sorted()}. (2) xTB MD seede les vitesses
    initiales sur l'horloge systeme (option \texttt{seed} ignoree dans le
    bloc \texttt{\$md}), comportement chaotique sur structures tendues ;
    abandon de la phase MD au profit d'une \textbf{perturbation z structuree
    deterministe} \(z_i' = z_i + 0{,}05\sin(2\pi i / N + 0{,}5)\) suivie de
    \texttt{xtb -{}-opt tight}. Avec \texttt{OMP\_NUM\_THREADS=1} +
    \texttt{MKL\_NUM\_THREADS=1}, le pipeline est byte-deterministe : md5 identique
    sur 5 runs (verifie sur 4 cas borderline h6). Trade-off accepte : on perd
    l'exploration thermique mais on gagne la reproductibilite -- alignement
    avec la demande des chimistes (pas d'analyse statistique). \\
  Mai 2026 (fin de mois) &
    \textbf{Refonte SQLite + Flask du viewer : \texttt{csp\_viewer/}}. Le
    \texttt{view.html} monolithique ne passe pas h9 (1{,}5~M solutions, 600~MB
    de JSON par config, navigateur plante). Backend SQLite multi-datasets
    (db\_all.db, PK \texttt{(h, config, mol)}) + serveur Flask local + SPA
    avec selecteur de dataset, lazy-loading, pagination. \textbf{Decouverte
    sur h9} : les "sols vides" apres run cluster ne sont pas des timeouts,
    mais des solutions \emph{combinatoirement valides mais geometriquement
    infaisables} (pentagone demande sur hexagone trop contraint, pattern
    \texttt{(1,1,1,1,1,0)}). \texttt{main.py} les saute proprement via
    \texttt{ValueError} attrape. Schema DB etendu avec \texttt{verdict}
    $\in$ \{\emph{plan, non\_plan, geom\_infeasible, xtb\_failed}\} et compteurs
    par categorie. Sur h9/no-freeze\_no-table, jusqu'a 75\% de
    \emph{geom\_infeasible} sur certaines mols --- ce n'est pas un manque de
    calcul. \textbf{Module \texttt{molviz/}} : visualisation 3D via 3Dmol.js,
    matching Kekule (Edmonds blossom) pour assigner les doubles, identification
    des radicaux. Filtrage anti-chord-bonds parasites : enumere tous les
    cycles simples $\leq 7$, garde uniquement les 5/6/7, supprime les bonds
    qui n'appartiennent a aucun cycle retenu. Resout le cas du faux 3-cycle
    visible sur les structures tres tendues (4 pentagones + 4 heptagones d'h9). \\
  \bottomrule
\end{tabular}
\end{center}
