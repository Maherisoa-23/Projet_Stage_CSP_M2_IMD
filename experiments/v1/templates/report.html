Methodologie (pipeline MD, parametres xTB, test de planarite)

Methode de validation arretee pour ce dashboard : MD courte + optimisation finale, selon le protocole recommande par les chimistes (Yannick Carissan). La dynamique moleculaire courte permet de casser une planarite parasite imposee par la reconstruction 2D avant l'optimisation, et de tester si la vraie geometrie minimum reste plate.

Pipeline

.graph (BenzAI DB) → Solveur CSP (ACE) → Reconstruction 3D → xtb --md (1 ps, 298 K) → xtb --opt tight (frame finale) → Test planarite (ACP)

Etape 1 : Solveur CSP

SolveurPyCSP3 + ACE (backend Java)
VariablesUne par cycle (taille ∈ {5, 6, 7})
ContraintesConservation des atomes, somme des cycles, optionnel : freeze des cycles imposes, table d'adjacence, regle 5/7
Configurations testees2³ = 8 combinaisons (--no-freeze, --no-table, --adj-57)

Etape 2 : Reconstruction 3D

PrincipePlacement geometrique des atomes a partir de la decomposition en cycles
Sortiesource.xyz dans solutions/sol_X/ (geometrie 2D plane par construction)

Etape 3 : MD courte (xtb --md)

Methode quantiqueGFN2-xTB (semi-empirique tight-binding etendu)
EnsembleNVT, thermostat de Berendsen
Temperature298.15 K
Duree1 ps
Pas d'integration4 fs (avec hmass=4 pour permettre ce pas)
Frequence de dump50 fs (~20 frames sur la trajectoire)
Velocites initialesgenerees par xTB (graine basee sur l'horloge -- voir note)
DeterminismeOMP_NUM_THREADS=1 + MKL_NUM_THREADS=1 → variance sub-seuil (~0.01°)
Sortiemd_traj.xyz (trajectoire), md_geom.xyz (input)

Etape 4 : Optimisation finale (xtb --opt)

InputDerniere frame de md_traj.xyz
Niveau d'optimisation--opt tight (convergence serree)
Sortiemd_final_opt.xyz -- c'est cette geometrie qui passe le test ACP
Timeout300 secondes par solution

Etape 5 : Test de planarite (ACP)

MethodeAnalyse en Composantes Principales sur les coordonnees atomiques (carbones uniquement)
MetriqueAngle maximal de deviation par rapport au plan moyen (3e CP)
Seuil10° — angle < seuil → plan, sinon non plan

Verdict (binaire MD)

🟢 Planangle ACP final < 10° sur md_final_opt.xyz
⚫ Non planangle ACP final ≥ 10°
Note non-determinisme : xTB seede ses velocites initiales sur l'horloge ; deux executions identiques peuvent donner des trajectoires legerement differentes. Avec OMP_NUM_THREADS=1 + MKL_NUM_THREADS=1 (active par defaut via --method md), la variance des angles finaux reste sub-seuil (typiquement < 0.01°), donc les verdicts plan / non plan sont stables.
